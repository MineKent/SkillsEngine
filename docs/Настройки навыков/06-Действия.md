# Actions — что сделать

`actions` — список строк внутри `groups`. Каждая строка — одна команда движка.

Важно:
- `msg`, `ab`, `title` применяются к **player-целям** текущей группы (`target`). Если цель не игрок — сообщение не отправится.

## Синтаксис

### `dmg <amount>`

Нанести урон целям.

- `amount` (обязательно) — количество урона.

```yml
actions:
  - "dmg 2"
```

### `heal <amount>`

Вылечить цели.

- `amount` (обязательно) — количество здоровья.

```yml
actions:
  - "heal 4"
```

### `potion <EFFECT> dur <sec> amp <n> [hide] [icon]`

Выдать эффект.

- `EFFECT` (обязательно) — тип эффекта (например, SPEED, SLOW, POISON).
- `dur <sec>` (обязательно) — длительность в секундах.
- `amp <n>` (обязательно) — усиление (0 = уровень I, 1 = уровень II, ...).
- `hide` (опционально) — отключить частицы ("пузырьки").
- `icon` (опционально) — скрыть иконку эффекта.

```yml
actions:
  - "potion SPEED dur 5 amp 1 hide icon"
```

### `wait <ticks>`

Подождать указанное количество тиков перед выполнением следующих действий.

- `ticks` (обязательно) — количество тиков ожидания (20 тиков = 1 секунда).

```yml
actions:
  - "wait 20"
```

### `lighting`

Ударить молнией по целям.

```yml
actions:
  - "lighting"
```

### `dash <strength> [x<value>] [y<value>] [flags(look)]`

Рывок вперёд.

- `<strength>` — общий множитель силы.
- `x<value>` (опционально) — горизонтальная скорость (умножается на `<strength>`).
- `y<value>` (опционально) — вертикальная скорость (умножается на `<strength>`).
  - если `y` не указать — вертикальная скорость сохраняется.
- `flags(look)` (опционально) — если указан, рывок делается **в сторону точки взгляда** (вектор от игрока к lookpoint), а не просто вперёд по направлению взгляда.

```yml
actions:
  - "dash 1.2 x 1.0 y 0.2"
```

### `particles <PARTICLE> [count <n>] [offset <d>] [speed <s>]`

Показать частицы в точке текущей группы.

- `PARTICLE` (обязательно) — Particle.
- `count <n>` (опционально) — количество.
- `offset <d>` (опционально) — разброс по осям.
- `speed <s>` (опционально) — скорость.

```yml
actions:
  - "particles CRIT count 20 offset 0.3"
```

### `sound <SOUND> vol <v> pitch <p> [mode <WORLD|CASTER|TARGETS>]`

Проиграть звук.

- `SOUND` (обязательно) — Sound.
- `vol <v>` (обязательно) — громкость.
- `pitch <p>` (обязательно) — тон.
- `mode <WORLD|CASTER|TARGETS>` (опционально) — где проигрывать:
  - `WORLD` — всем рядом.
  - `CASTER` — только кастеру.
  - `TARGETS` — целям группы.

```yml
actions:
  - "sound ENTITY_PLAYER_LEVELUP vol 1 pitch 1 mode CASTER"
```

### `msg <text...>`

Сообщение в чат (**отключено**).

- `text...` — текст сообщения.

```yml
actions:
  - "msg &7(чат отключён)"
```

### `ab <text...>`

Actionbar.

- `text...` — текст.

```yml
actions:
  - "ab &aActionbar"
```

### `title <title> sub <subtitle...> in <t> stay <t> out <t>`

Title/subtitle.

- `<title>` (обязательно) — заголовок.
- `sub <subtitle...>` (обязательно) — подзаголовок.
- `in <t>` (обязательно) — время появления (тики).
- `stay <t>` (обязательно) — время показа (тики).
- `out <t>` (обязательно) — время исчезновения (тики).

```yml
actions:
  - "title &eTitle sub &7Sub in 10 stay 40 out 10"
```

### `cmd [console|player] <command...>`

Выполнить команду.

- `console|player` (опционально) — от чьего имени выполнять.
- `command...` (обязательно) — команда без `/`.

```yml
actions:
  - "cmd console say cast {skill} by {player}"
```

### `tp <caster|target|center>`

Телепортировать кастера.

- `caster|target|center` (обязательно) — куда телепортировать:
  - `caster` — в позицию кастера.
  - `target` — в позицию цели.
  - `center` — в `targetLocation` текущей группы (например, `lookpoint` / `area ... look ...`).

```yml
actions:
  - "tp center"
```

### `kb <strength> [x<value>] [y<value>] [flags(look)]`

Оттолкнуть цели.

- `<strength>` — общий множитель силы.
- `x<value>` — горизонтальная скорость (умножается на `<strength>`).
- `y<value>` — вертикальная скорость (умножается на `<strength>`).
- `flags(look)` (опционально) — если указан, толчок считается **от точки взгляда**, а не от кастера.

```yml
actions:
  - "kb 1.0 x1.2 y0.2 flags(look)"
```

### `pull <strength> [x<value>] [y<value>] [flags(look)]`

Притянуть цели.

- `<strength>` — общий множитель силы.
- `x<value>` — горизонтальная скорость (умножается на `<strength>`).
- `y<value>` — вертикальная скорость (умножается на `<strength>`).
- `flags(look)` (опционально) — если указан, притяжение считается **к точке взгляда**, а не к кастеру.

```yml
actions:
  - "pull 1.0 x1.2 y0.2 flags(look)"
```

### `fire <seconds>`

Поджечь цели.

- `seconds` (обязательно) — длительность горения.

```yml
actions:
  - "fire 3"
```

### `explosion [power] [fire] [break] [damage]`

Создать взрыв в точке текущей группы.

- `power` (опционально) — сила взрыва.
- `fire` (опционально) — поджигать.
- `break` (опционально) — ломать блоки.
- `damage` (опционально) — наносить урон.

```yml
actions:
  - "explosion 2 fire break damage"
```

### `give <MATERIAL> [amount]`

Выдать предмет кастеру.

- `MATERIAL` (обязательно) — название предмета (например, EMERALD).
- `amount` (опционально) — количество.

```yml
actions:
  - "give EMERALD 1"
```

### `take <MATERIAL> [amount]`

Забрать предмет у кастера.

- `MATERIAL` (обязательно) — название предмета (например, EMERALD).
- `amount` (опционально) — количество.

```yml
actions:
  - "take EMERALD 1"
```

### `spawn <ENTITY> [count]`

Заспавнить сущность.

- `ENTITY` (обязательно) — тип сущности (например, ZOMBIE).
- `count` (опционально) — количество.
  - если `target: area ...` и в области найдены цели — спавн распределяется по целям в области.
  - если нужно заспавнить **в точке**, куда смотришь (а не в сущностях рядом) — используй `target: "lookpoint ..."`.

```yml
actions:
  - "spawn ZOMBIE 1"
```

## Плейсхолдеры

- `{player}`
- `{skill}`
- `{target}`
