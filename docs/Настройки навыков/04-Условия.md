# Условия (`conditions`)

Условия определяют, *можно ли* кастовать навык.

Формат:

```yml
conditions:
  - type: HAS_PERMISSION
    permission: "my.perm"
```

`conditions` — список объектов, где у каждого есть поле `type`.

## Поддерживаемые условия

### `HAS_PERMISSION`

```yml
- type: HAS_PERMISSION
  permission: "skills.my"
```

### `LEVEL_AT_LEAST`
Проверяет ванильный уровень игрока (`Player#getLevel`).

```yml
- type: LEVEL_AT_LEAST
  min: 10
```

Поля:

- `min: number` — минимальный уровень (можно также `level: 10`)

### `HAS_ITEM`

```yml
- type: HAS_ITEM
  material: "EMERALD"
  amount: 2
  where: INVENTORY
```

Поля:

- `material: string` — материал Bukkit (обязательно)
- `amount: number` — количество (по умолчанию 1)
- `where: INVENTORY|HAND|MAIN_HAND` — где искать (по умолчанию `INVENTORY`)

### `WORLD_ALLOWED`
Разрешённые/запрещённые миры.

```yml
- type: WORLD_ALLOWED
  allowed: ["world", "world_nether"]
  denied: ["arena"]
```

Оба списка опциональны.

## Про кулдаун

`cooldown` проверяется автоматически (и не является condition).

`COOLDOWN_READY` считается deprecated — оставлено только для совместимости, плагин выведет предупреждение.
