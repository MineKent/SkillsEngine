# Actions — что сделать

`actions` — список строк внутри `groups`. Каждая строка — одна команда движка.

### Как это работает

* Действия выполняются для целей, выбранных параметром `target` текущей группы.
* Действия выполняются по порядку, сверху вниз.

> \[!TIP] Если нужно сделать "комбо" из нескольких этапов — используйте несколько `groups` с разным `target`.

## Основные действия

### `dmg <amount>`

Наносит урон целям.

* `amount` (обязательно): количество урона.

```yml
actions:
  - "dmg 2"
```

### `heal <amount>`

Лечит цели.

* `amount` (обязательно): количество здоровья.

```yml
actions:
  - "heal 4"
```

### `potion <EFFECT> dur <sec> amp <n> [hide] [icon]`

Выдает эффект зелья.

* `EFFECT`: тип эффекта (`SPEED`, `SLOW`, `POISON`).
* `dur <sec>`: длительность в секундах.
* `amp <n>`: усиление (0 = уровень I, 1 = уровень II, ...).
* `hide` (опционально): отключает частицы ("пузырьки").
* `icon` (опционально): скрывает иконку эффекта в интерфейсе.

```yml
actions:
  - "potion SPEED dur 5 amp 1"
```

### `set_fire <seconds>`

Поджигает цели.

* `seconds` (обязательно): длительность горения в секундах.

```yml
actions:
  - "fire 3"
```

## Передвижение и позиционирование

### `dash <strength> [x <val>] [y <val>] [look]`

Рывок для кастера.

* `strength`: общий множитель силы.
* `x <val>` (опционально): горизонтальная скорость.
* `y <val>` (опционально): вертикальная скорость. Если `y` не указан, вертикальная скорость сохраняется.
* `look` (опционально): если указан, рывок делается в сторону точки взгляда, а не по направлению взгляда.

```yml
actions:
  - "dash 1.2 x 1.0 y 0.2"
```

### `knockback <strength> [x <val>] [y <val>] [look]`

Отталкивает цели от кастера.

* `strength`, `x`, `y`, `look`: аналогично `dash`.

```yml
actions:
  - "kb 1.0 x 1.2 y 0.2"
```

### `pull <strength> [x <val>] [y <val>] [look]`

Притягивает цели к кастеру.

* `strength`, `x`, `y`, `look`: аналогично `dash`.

```yml
actions:
  - "pull 1.0 x 1.2 y 0.2"
```

### `teleport <caster|target|center|location>`

Телепортирует кастера.

* `caster|target|center`: куда телепортировать. `center` — это `targetLocation` текущей группы (например, `lookpoint`).
* `location`: телепорт по координатам (`world`, `x`, `y`, `z`, `yaw`, `pitch`).

```yml
actions:
  - "teleport center"
  - "teleport location world world_the_end x 0 y 100 z 0"
```

## Взаимодействие с миром

### `setblock <MATERIAL> [form] [args...]`

Устанавливает блоки в точке, указанной `target`.

* `MATERIAL`: материал блока (`STONE`, `GLASS`).
* `form` (опционально): форма для постройки. По умолчанию `single`.
  * `single`: один блок.
  * `sphere`: сфера. Аргумент: `radius <r>`.
  * `wall`: стена. Аргументы: `width <w> height <h>`.
  * `box`: полая коробка. Аргументы: `width <w> height <h> depth <d>`.

```yml
actions:
  - "setblock GLASS form sphere radius 3"
  - "setblock BRICKS form wall width 10 height 5"
```

### `grow [radius]`

Ускоряет рост растений (применяет эффект костной муки).

* `radius` (опционально): радиус применения. Если не указан, действует на один блок.

```yml
actions:
  - "grow 3"
```

### `var set <name> = <value>`

Устанавливает значение переменной.

```yml
actions:
  - "var set rage = 0"
```

### `var modify <name> <op> <value>`

Изменяет числовую переменную.

```yml
actions:
  - "var modify rage add 10"
```

### `var toggle <name>`

Переключает булеву переменную.

```yml
actions:
  - "var toggle stealth_mode"
```

### `var unset <name>`

Удаляет переменную.

```yml
actions:
  - "var unset rage"
```

### `drop <MATERIAL> [amount]`

Выбрасывает предмет в точке, указанной `target`.

* `MATERIAL`: материал предмета.
* `amount` (опционально): количество. По умолчанию 1.

```yml
actions:
  - "drop DIAMOND 5"
```

### `explosion [power] [fire] [break] [damage]`

Создает взрыв.

* `power` (опционально): сила взрыва.
* `fire` (опционально): поджигать блоки.
* `break` (опционально): ломать блоки.
* `damage` (опционально): наносить урон.

```yml
actions:
  - "explosion 2 fire break damage"
```

### `lightning`

Ударяет молнией по целям или в указанную `targetLocation`.

```yml
actions:
  - "lightning"
```

### `spawn <ENTITY> [count]`

Спавнит одну или несколько сущностей.

* `ENTITY`: тип сущности (`ZOMBIE`, `VILLAGER`).
* `count` (опционально): количество.

```yml
actions:
  - "spawn ZOMBIE 5"
```

## Инвентарь

### `give_item <MATERIAL> [amount]`

Выдает предмет кастеру.

* `MATERIAL`: материал предмета.
* `amount` (опционально): количество.

```yml
actions:
  - "give_item EMERALD 1"
```

### `take_item <MATERIAL> [amount]`

Забирает предмет у кастера.

* `MATERIAL`: материал предмета.
* `amount` (опционально): количество.

```yml
actions:
  - "take_item EMERALD 1"
```

### `equip <MATERIAL> <slot>`

Надевает предмет на цели.

* `MATERIAL`: материал (`DIAMOND_HELMET`).
* `slot`: слот для экипировки (`head`, `chest`, `legs`, `feet`, `hand`, `offhand`).

```yml
actions:
  - "equip PUMPKIN head"
```

### `unequip <slot>`

Снимает предмет с указанного слота у целей.

* `slot`: слот для очистки.

```yml
actions:
  - "unequip head"
```

## Служебные действия

### `wait <ticks>`

Приостанавливает выполнение цепочки действий. **Работает только в асинхронном режиме.**

* `ticks`: количество тиков для ожидания (20 тиков = 1 секунда).

```yml
actions:
  - "msg &aПодождите..."
  - "wait 40"
  - "msg &aСпасибо, что подождали!"
```

### `command [console|player] <command...>`

Выполняет команду.

* `console|player` (опционально): от чьего имени выполнять (по умолчанию `console`).
* `command...`: команда без `/`.

```yml
actions:
  - "cmd console say Игрок {player} использовал умение {skill}!"
```

### `msg`, `ab`, `title`

Отправляют сообщения целям (если цель — игрок).

* `msg <text...>`: сообщение в чат.
* `ab <text...>`: сообщение в action bar.
* `title <title> sub <subtitle>`: заголовок и подзаголовок.

```yml
actions:
  - "msg &aВам пришло сообщение!"
  - "ab &eAction bar!"
  - "title &6Заголовок sub &7Подзаголовок"
```

## Визуальные и звуковые эффекты

### `particles <PARTICLE> [count] [offset] [speed]`

Показывает частицы.

* `PARTICLE`: название частиц (`CRIT`, `FLAME`).
* `count` (опционально): количество.
* `offset` (опционально): разброс.
* `speed` (опционально): скорость.

```yml
actions:
  - "particles CRIT count 20 offset 0.3"
```

### `sound <SOUND> vol <v> pitch <p> [mode]`

Проигрывает звук.

* `SOUND`: название звука (`ENTITY_PLAYER_LEVELUP`).
* `vol <v>`: громкость (от 0.0 до 1.0).
* `pitch <p>`: тон (от 0.0 до 2.0).
* `mode` (опционально): `WORLD` (для всех), `CASTER` (для кастера), `TARGETS` (для целей).

```yml
actions:
  - "sound ENTITY_PLAYER_LEVELUP vol 1 pitch 1.2 mode CASTER"
```

## Плейсхолдеры

В текстовых полях (например, `msg` или `cmd`) можно использовать плейсхолдеры:

* `{player}` — имя кастера.
* `{skill}` — ID умения.
* `{target}` — имя цели (если она есть).
