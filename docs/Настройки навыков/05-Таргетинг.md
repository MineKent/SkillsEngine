# Target — кого выбрать

`target` задаётся внутри `groups`.

Важно:
- Если в текущей группе цель не найдена (например, `target` и ты ни в кого не целишься), то **эта группа пропускается**, а остальные группы всё равно выполняются.

## Синтаксис

- `caster` — цель = кастер.
- `target [range]` — одна цель по взгляду с лимитом дистанции.
  - `range` опционален.
  - Для триггера `hit` можно не указывать `range` (цель уже известна из события удара).
- `lookpoint [maxBlocks] [air]` — точка в мире по лучу взгляда кастера (**без поиска сущностей рядом**).
  - `maxBlocks` опционален (если не указан — дальность практически не ограничена).
  - Если `air` не указан и в пределах `maxBlocks` нет блока — группа пропускается (как `NO_TARGET`).
- `area <radius> [caster|target]` — все цели в радиусе от точки.
- `area <radius> look [maxBlocks] [air]` — центр области по лучу взгляда кастера.
  - Если `air` не указан и в пределах `maxBlocks` нет блока — группа пропускается (как `NO_TARGET`).

## Примеры

```yml
groups:
  - target: "caster"
    actions:
      - "ab caster"

  - target: "target 5"
    actions:
      - "dmg 4"

  - target: "lookpoint 20 air"
    actions:
      - "particles CRIT count 10 offset 0.2"

  - target: "area 4 caster"
    actions:
      - "dmg 2"

  - target: "area 6 target"
    actions:
      - "particles CRIT count 30 offset 0.4"

  - target: "area 6 look 20 air"
    actions:
      - "particles CRIT count 30 offset 0.4"
```
