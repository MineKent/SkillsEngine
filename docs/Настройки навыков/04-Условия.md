# Условия (`conditions`)

Условия определяют, *можно ли* кастовать навык.

`conditions` — список. Каждый элемент может быть:

- объектом YAML с `type: ...`
- строкой DSL

## DSL-формат

```yml
conditions:
  - "perm skills.smite"
  - "lvl 10"
  - "world allow world world_nether"
  - "item EMERALD 2 where inv"
```

Поддерживаемые DSL условия:

- `perm <node>` — permission
- `lvl <min>` — ванильный уровень игрока (`Player#getLevel`)
- `item <MAT> <AMOUNT> [where inv|hand]` — наличие предмета
- `world allow <w1> <w2>...` — разрешённые миры
- `world deny <w1> <w2>...` — запрещённые миры

## Классический YAML-формат

```yml
conditions:
  - type: HAS_PERMISSION
    permission: "skills.my"
  - type: LEVEL_AT_LEAST
    min: 10
```

## Про кулдаун

`cooldown` проверяется автоматически (и не является condition).

`COOLDOWN_READY` считается deprecated — оставлено только для совместимости, плагин выведет предупреждение.
