# Навык — общий вид

Файлы с умениями находятся в папке `plugins/SkillsEngine/skills/` и имеют формат `.yml`.

### Структура файла

- Один `.yml` файл может содержать **неограниченное количество умений**.
- Каждое умение определяется **ключом (названием) верхнего уровня**, который является его уникальным `ID`.

### Пример файла с двумя умениями

```yml
# ID первого умения
fireball:
  trigger: "cmd fireball" # Как активировать
  cooldown: 5s
  groups:
    - target: "target 5"
      actions:
        - "fire 3"

# ID второго умения
heal:
  trigger: "rc" # Right-Click
  conditions:
    - "lvl 10"
  cost:
    - "mana 20"
  groups:
    - target: "caster"
      actions:
        - "heal 10"
```

## Поля умения

- `trigger` — (обязательно) список триггеров, которые активируют умение.
- `groups` — (обязательно) список групп действий. В умении должна быть хотя бы одна группа.
- `conditions` — список условий, при которых умение можно использовать.
- `cost` — список затрат на использование (мана, опыт, предметы).
- `cooldown` — задержка между использованиями.
- `denyMessage` — сообщение, которое получит игрок, если использование умения не удалось (например, из-за нехватки ресурсов).

### `groups`

Каждая группа содержит:
- `target` — кого выбрать в качестве цели.
- `actions` — что сделать с выбранными целями.

**Важно**: если `target` группы не находит ни одной цели, то вся группа действий пропускается.

Пример группы:

```yml
groups:
  - target: "caster"
    actions:
      - "msg &aВы исцелили себя!"
      - "sound ENTITY_PLAYER_LEVELUP vol 1 pitch 1"
```
