# conditions — когда запретить каст

`conditions` — **список строк**, каждая строка — отдельная проверка. Если хотя бы одна проверка не проходит, каст отклоняется.

> Важно: строки условий лучше всегда брать в кавычки (`"..."`), чтобы YAML не ломался на пробелах/двоеточиях.

## Синтаксис

- `perm <node>` — требуется право (permission node).
  - `<node>` — строка, например `skills.smite`.

- `lvl <min>` — требуется минимум указанный уровень.
  - `<min>` — целое число.
  - Уровень берётся из ванильного уровня Minecraft (`Player#getLevel`).

- `item <MAT> <AMOUNT> [where inv|hand]` — требуется предмет.
  - `<MAT>` — материал Bukkit (`Material`), например `EMERALD`, `DIAMOND_SWORD`.
  - `<AMOUNT>` — количество (целое число).
  - `where inv` — проверять наличие в инвентаре.
  - `where hand` — проверять предмет в руке.
  - Если `where ...` не указано — по умолчанию используется `inv`.

- `world allow <w1> <w2>...` — каст разрешён **только** в перечисленных мирах.
  - `<w1> <w2>...` — имена миров (как в папке `world/`, `world_nether/` и т.п.).

- `world deny <w1> <w2>...` — каст запрещён в перечисленных мирах.
  - `<w1> <w2>...` — имена миров.

## Примеры

```yml
conditions:
  - "perm skills.smite"
  - "lvl 10"
  - "world deny arena"
  - "item EMERALD 2 where inv"
```

Пример для нескольких миров:

```yml
conditions:
  - "world allow world world_nether"
```

Кулдаун задаётся отдельно: `cooldown: 5s`.
